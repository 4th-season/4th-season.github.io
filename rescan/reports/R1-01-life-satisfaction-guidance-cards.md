# R1-01 삶의 만족도 조건형 결과 카드 설계 보고서

- 대상 체크리스트: `life-satisfaction`
- 데이터: `rescan/data/guidance/life-satisfaction.json`
- 작성일: 2026-07-24
- 상태: 1차 집필 완료, 실행 엔진 연결 전

## 목적

결과 하단에 일반 행동 여러 개와 일반 성찰 질문 여러 개를 자동으로 나열하지 않는다. 응답에서 확인되는 생활 장면을 근거로 카드 한 장만 선택하고, 해당 카드에는 장면 해설 뒤 행동 실험·성찰 질문·추가 해설 중 하나만 둔다.

## 구성

- 생활의 안정: 4장
- 관계와 소속감: 4장
- 활동과 역할: 4장
- 방향과 의미: 4장
- 합계: 16장

## 선택 원칙

1. 영역별 환산 점수가 가장 높은 영역을 먼저 본다.
2. 해당 영역에서 원응답 조건이 모두 맞는 카드만 후보로 삼는다.
3. 여러 카드가 맞으면 조건이 더 구체적인 카드와 우선도가 높은 카드를 선택한다.
4. 결과 화면에는 카드 한 장만 출력한다.
5. 적합한 카드가 없으면 일반 문장으로 빈칸을 채우지 않는다.
6. AI가 실시간으로 행동이나 질문을 생성하지 않는다.

## 집필 원칙

- 총점만으로 생활 장면을 단정하지 않는다.
- 시간 관리, 의지, 긍정성 부족으로 문제를 환원하지 않는다.
- 관계가 어렵다는 응답에 만남 확대나 연락을 자동 권하지 않는다.
- 방향이 흐린 응답에 목표 설정을 자동 권하지 않는다.
- 활동이 부족한 응답에 취미 추가를 자동 권하지 않는다.
- 행동 카드는 성공 여부보다 반응 관찰을 중심으로 쓴다.
- 질문 카드는 서로 어긋나는 응답이나 유지되는 자원과 부담 사이의 긴장을 드러낸다.
- 심한 생활 부담이 보이면 실천 과제보다 기본 조건과 지원 연결을 먼저 설명한다.

## 카드 출력 유형

### `experiment`
특정 생활 장면에서 한 번 시험하고 반응을 관찰할 수 있을 때만 사용한다.

### `question`
바로 행동을 권하기보다 응답 사이의 긴장을 오래 생각해 볼 필요가 있을 때 사용한다.

### `scene`
행동이나 질문을 덧붙이는 것이 오히려 부담이 될 때 추가 해설만 제공한다.

## 현재 범위

이번 단계에서는 16개 카드의 문장과 선택 조건만 확정했다. 다음 단계에서 카드 데이터 로딩, 영역 우선순위 판정, 조건 비교, 결과 화면 단일 카드 출력, 기존 질문·행동 나열 억제 기능을 구현한다.
