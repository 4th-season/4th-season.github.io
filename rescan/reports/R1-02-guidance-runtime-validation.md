# R1-02 삶의 만족도 조건형 해설 실행 검증

- 기준일: 2026-07-24
- 대상 체크리스트: `life-satisfaction`
- 조건 카드: `rescan/data/guidance/life-satisfaction.json`
- 선택 엔진: `rescan/assets/js/guidance-engine.js`
- 화면 연결: `rescan/assets/js/guidance-ui.js`
- 브라우저 QA: `rescan/qa/guidance-life-satisfaction.html`

## 구현 내용

1. 원응답과 기존 Re:Scan 채점 결과를 함께 사용한다.
2. 영역별 환산 점수가 가장 높은 영역만 카드 후보로 삼는다.
3. 카드의 `all`·`any`·`none` 조건을 원응답에 직접 대조한다.
4. 조건 문항 수가 많은 카드를 우선하고, 다음으로 카드 우선순위를 비교한다.
5. 최상위 후보가 동일 조건 수·동일 우선순위로 겹치면 임의 선택하지 않고 카드 출력을 생략한다.
6. 조건형 해설 데이터가 연결된 체크리스트에서는 기존 일반 질문 목록과 일반 행동 목록을 제거한다.
7. 선택된 카드는 장면 해설과 `scene`·`question`·`experiment` 중 하나의 응답만 표시한다.

## 끼워 맞추기 방지 판정

- 총점만으로 문장을 고르지 않음
- 특정 영역만 높다는 이유로 일반 조언을 자동 생성하지 않음
- 조건에 맞는 카드가 없으면 질문과 행동을 출력하지 않음
- 동률 후보를 배열 순서로 임의 선택하지 않음
- 사용자 응답은 서버나 저장소로 전송하지 않고 현재 브라우저에서만 카드 선택에 사용함
