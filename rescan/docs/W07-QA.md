## 공통 엔진 확인

- 6개 항목 모두 `index.json`에서 `published` 상태와 실제 `dataPath`를 가짐
- 6개 데이터 파일 모두 id, title, subtitle, description, period, scale, questions, 결과 구간을 포함
- 정식 점검 주소는 `/rescan/checklist.html?id={id}` 규칙을 공통 사용
- 문항 순서는 세션 안정형 영역 균형 배열을 사용하며 점수는 문항 id로 계산
- 다시 점검하기, 결과 인쇄·PDF 저장, 다른 주제 보기 기능을 공통 사용

## 점수 시나리오

- 각 12문항 주제의 최소 응답, 중간 응답, 최대 응답이 연속 결과 구간 안에 포함되는 구조 확인
- 역채점 문항은 공통 계산 엔진에서 척도 최솟값·최댓값을 기준으로 변환
- 관계경계 결과 범위 0~36점: 0~8 / 9~17 / 18~26 / 27~36으로 빈틈과 중복 없음

## 검색·분류

- 제목·부제·keywords 통합 검색 적용
- 전체, 정서, 사고, 자기관계, 관계, 생활, 일·학습 필터 적용
- 모바일에서는 필터를 가로 스크롤 방식으로 유지
- 검색 초기화 버튼과 결과 없음 안내 제공
- 실행 가능한 공개 항목만 링크로 제공하도록 구성

## 메타·색인

- Re:Scan 홈 canonical과 Open Graph 정보 적용
- 체크리스트는 id별 canonical, 제목, 설명, URL을 동적으로 적용
- id가 없거나 준비 상태인 화면은 noindex 유지
- 공개 체크리스트는 index,follow로 전환
- 공유 대표 이미지는 `/assets/map-note.png`의 절대 주소 사용
- sitemap.xml에 Re:Scan 홈, 안내, 6개 해설 글 반영

## 안전 문구

- 관계경계 결과는 상대방을 가해자로 단정하지 않음
- 관계 단절을 자동 해결책으로 제시하지 않음
- 응답자에게 관계의 모든 책임을 돌리지 않음
- 위협·강압·폭력·스토킹은 일반 경계 점검과 분리하여 전문 지원 안내
