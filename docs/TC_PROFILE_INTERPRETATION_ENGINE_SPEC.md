# 기질·성격 자기이해 점검 해석 엔진 명세

- 상태: 공개 후보 0.1
- 적용 모드: `profile`
- 실행 방식: 브라우저 내부 규칙 기반 조립
- AI API 및 토큰 사용: 없음

## 1. 보고서 구성 순서

1. 전체 요약 1개
2. 프로필 균형도 1개
3. 주요 2영역 조합 최대 2개
4. 7개 영역별 해석
5. 응답 참고도 1개
6. 비진단적 자기이해 도구 고지문

## 2. 기본 해석 모듈

7개 영역마다 개인 내부 상대 위치 5단계를 사용한다.

- `much_higher`
- `somewhat_higher`
- `similar`
- `somewhat_lower`
- `much_lower`

각 모듈은 다음 필드를 갖는다.

- `title`: 결과 카드 제목
- `summary`: 현재 경향 설명
- `strength`: 도움이 될 수 있는 장면
- `caution`: 부담이 될 수 있는 장면
- `question`: 자기점검 질문

총 35개 기본 영역 모듈을 둔다.

## 3. 상태명 연결

채점 엔진과 해석 데이터의 명칭 차이는 해석 엔진 내부에서 변환한다.

| 채점 상태 | 해석 상태 |
|---|---|
| clearly_higher | much_higher |
| somewhat_higher | somewhat_higher |
| near_average | similar |
| balanced | similar |
| somewhat_lower | somewhat_lower |
| clearly_lower | much_lower |

## 4. 조합 선택 규칙

- 조합 모듈은 우선순위가 높은 순서로 검사한다.
- 한 보고서에 최대 2개만 출력한다.
- 같은 영역 쌍의 조합을 중복 출력하지 않는다.
- 두 조합이 동일 영역만 반복할 경우 후순위 조합을 제외한다.
- 점수 차이가 작은 경우 전체 요약에서는 단일 우세 경향으로 단정하지 않는다.
- 조합 설명은 기본 영역 설명을 대체하지 않고 관계를 보충한다.

초기판에는 다음 핵심 관계를 우선 적용한다.

- 변화 반응 × 불확실성 민감성
- 변화 반응 × 지속성
- 불확실성 민감성 × 자기조절
- 관계 반응성 × 협력성
- 지속성 × 자기조절
- 협력성 × 의미지향

## 5. 중복 및 길이 제한

- 동일 ID 또는 동일 카드 유형·식별자 조합은 한 번만 사용한다.
- 전체 요약에서 사용한 조합도 상세 조합 카드로 한 번 더 보일 수 있으나, 화면 구현 단계에서 같은 문장을 반복하지 않도록 요약형과 상세형 표시를 구분한다.
- 결과 화면은 기본적으로 10~12개 주요 카드 안에서 구성한다.
- 모바일에서는 영역별 상세 내용을 접기·펼치기로 제공할 수 있다.

## 6. 해석 금지

- 사회적 평균, 백분위, 정상 범위로 표현하지 않는다.
- 성격 유형을 확정하지 않는다.
- 정신질환, 장애, 능력, 도덕성, 직업 적합성을 추론하지 않는다.
- 높은 점수와 낮은 점수를 우열로 표현하지 않는다.
- 조합 수가 많다는 이유로 정확도 또는 과학성을 주장하지 않는다.

## 7. 현재 모듈 규모

- 기본 영역 해석: 35개
- 균형도 해석: 4개
- 응답 참고도: 3개
- 주요 2영역 조합: 12개
- 총 데이터 모듈: 54개

이후 화면과 가상 응답 QA가 안정되면 2영역 조합을 40개 이상으로 확대하고 주요 3영역 조합을 별도 추가한다.

## 8. QA 기준

- 모든 영역·모든 상태에 해석 모듈이 존재해야 한다.
- 채점 상태가 해석 상태로 누락 없이 변환되어야 한다.
- 균형형 응답에서 7개 영역 모두 `similar` 해석을 사용해야 한다.
- 조합은 최대 2개를 넘지 않아야 한다.
- 모든 출력 섹션에 제목과 본문 또는 요약이 있어야 한다.
- 고지문은 모든 결과에 포함되어야 한다.
