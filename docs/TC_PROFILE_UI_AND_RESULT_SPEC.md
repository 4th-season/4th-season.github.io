# 기질·성격 자기이해 점검 화면·결과 명세

- 상태: 초기 공개판 구현 기준
- 대상 화면: `/rescan/profile.html`
- 문항 수: 70문항
- 진행 방식: 10문항 × 7단계
- 계산 방식: 브라우저 내부 규칙 기반 계산

## 1. 시작 화면

시작 화면에는 다음 내용을 표시한다.

- 초기 공개판·만 18세 이상 표시
- 70문항, 7개 영역, 예상 소요시간
- 검사 응답과 결과가 운영자 서버로 전송되지 않는다는 안내
- 정식 TCI, 표준화 심리검사 또는 전문 평가가 아니라는 고지
- 이용 범위 확인 체크박스
- 설계 원칙 연결

동의 체크 전에는 문항 화면으로 이동할 수 없다.

## 2. 단계형 문항 화면

- 한 단계에는 10문항만 표시한다.
- 문항은 7개 영역이 반복적으로 섞이도록 배열한다.
- 같은 세션에서는 문항 순서를 유지한다.
- 각 문항은 5점 척도로 응답한다.
- 현재 단계의 미응답이 있으면 다음 단계로 이동하지 않는다.
- 이전 단계로 돌아가도 기존 응답을 유지한다.
- 전체 응답 수와 진행률을 표시한다.
- 문항 영역명과 정·역방향 정보는 사용자에게 표시하지 않는다.

## 3. 임시 저장

장문 점검의 새로고침 복구를 위해 다음 정보만 현재 탭의 `sessionStorage`에 저장한다.

- 문항 ID 순서
- 현재 단계
- 문항별 응답값
- 이용 범위 확인 여부

응답은 외부 서버나 분석 서비스로 전송하지 않는다. 결과 화면에서 `처음부터 다시 하기`를 누르면 임시 저장 정보를 삭제한다.

## 4. 결과 화면 순서

1. 전체 요약
2. 7개 영역 점수 막대
3. 영역 간 차이와 균형도
4. 주요 2영역 조합 최대 2개
5. 7개 영역별 상세 해석
6. 응답 참고도
7. 비진단적 도구 고지
8. 인쇄·PDF, 초기화, 홈 이동

## 5. 점수 표시 원칙

- 각 영역은 0~100 범위로 환산한다.
- 점수는 사회적 평균, 백분위, 정상 범위를 뜻하지 않는다.
- 7개 영역은 모두 같은 길이의 막대로 비교한다.
- 점수의 높고 낮음을 우수·열등, 건강·위험으로 표현하지 않는다.
- 영역별 결과는 가능성·상황·생활 조건과 함께 설명한다.

## 6. 영역별 상세 카드

각 영역은 가능한 경우 다음 내용을 포함한다.

- 현재 경향 요약
- 도움이 될 수 있는 장면
- 부담이 될 수 있는 장면
- 함께 살펴볼 환경이나 조건
- 자기점검 질문

해석 데이터에 해당 필드가 없으면 빈 카드를 만들지 않는다.

## 7. 모바일 기준

- 768px 이하에서는 응답 선택지를 한 열로 표시한다.
- 결과 점수 막대는 라벨과 점수를 첫 줄, 막대를 둘째 줄에 배치한다.
- 단계 이동 버튼은 한 열로 표시한다.
- 고정 진행 바가 헤더와 겹치지 않아야 한다.
- 최소 지원 폭은 320px이다.

## 8. 접근성 기준

- 모든 문항은 `fieldset`과 `legend`를 사용한다.
- 라디오 응답은 키보드로 선택할 수 있어야 한다.
- 진행 메시지와 오류 메시지는 보조기술에 전달한다.
- 색상만으로 선택 상태나 점수를 전달하지 않는다.
- `prefers-reduced-motion`에서 불필요한 전환을 제거한다.

## 9. 인쇄·PDF 기준

- 헤더, 푸터, 버튼, 시작 안내는 인쇄하지 않는다.
- 전체 요약, 점수 막대, 조합과 영역별 해석을 포함한다.
- 결과 블록은 가능한 한 페이지 중간에서 잘리지 않게 한다.
- PDF 파일의 보관과 삭제 책임은 사용자에게 있음을 알린다.

## 10. 공개 전 QA

### 진행

- 시작 동의 체크가 작동하는가
- 7단계가 정확히 표시되는가
- 단계별 문항 수가 10개인가
- 미응답 차단 후 첫 미응답 문항으로 이동하는가
- 이전 단계 응답이 유지되는가
- 새로고침 후 진행 상태가 복원되는가
- 초기화 후 모든 응답과 문항 순서가 다시 생성되는가

### 계산과 결과

- 70개 응답 완료 후에만 결과가 생성되는가
- 7개 영역 점수가 모두 출력되는가
- 조합 결과가 최대 2개인가
- 동일 응답에서 참고도 안내가 표시되는가
- 결과문에 빈 제목이나 빈 본문이 없는가
- 점수와 해석 영역이 서로 일치하는가

### 화면

- 320px, 360px, 768px, 1024px, 데스크톱에서 확인
- 긴 문항과 긴 결과 제목이 넘치지 않는가
- 키보드만으로 전 과정 수행 가능한가
- 인쇄 미리보기에서 버튼과 내비게이션이 제외되는가

## 11. 현재 공개 상태

`profile.html`은 구현되었으나 Re:Scan 홈의 공개 목록에는 아직 연결하지 않는다. 화면 QA와 문장 검수를 마친 뒤 후보 데이터 상태를 `published`로 변경하고 홈 카드와 연결한다.
