# 기질·성격 자기이해 점검 — 공개형 개발 기준

## 1. 공개 방식

이 프로그램은 별도 검증용 표본을 수집하지 않고 공개한다. 따라서 첫 공개본부터 다음 표현을 고정한다.

- 공개 상태: `베타` 또는 `초기 공개판`
- 성격: 자기이해용 간이 점검 도구
- 비해당: 정식 TCI, 표준화 심리검사, 임상검사, 진단도구
- 적용 대상: 만 18세 이상 성인
- 이용 금지: 채용, 인사평가, 입학, 치료 결정, 성격장애 판정, 개인의 능력·가치 판단

공개 시점은 문항의 통계적 검증이 아니라 내부 설계 검토, 코드 검증, 접근성 확인, 결과 문장 충돌 검사까지 완료한 뒤로 한다.

## 2. 공개 화면 고지

### 시작 화면

> 이 점검은 일상에서 나타나는 반응 방식과 성격 경향을 돌아보기 위해 독자적으로 구성한 자기이해용 간이 도구입니다. 정식 TCI 검사나 표준화된 심리검사가 아니며, 정신건강 상태나 성격장애를 진단하지 않습니다. 결과는 현재 응답을 바탕으로 구성되는 참고 자료로만 활용해 주세요.

### 결과 화면

> 이 결과는 검증된 인구 규준에 따른 판정이 아닙니다. 점수는 이번 응답 안에서 나타난 상대적인 경향을 보여주며, 높은 점수와 낮은 점수는 우열·정상·비정상을 의미하지 않습니다.

### 하단 고정문

> 보다 정확하고 심층적인 평가가 필요한 경우에는 관련 자격을 갖춘 전문가 또는 정식 검사 제공기관의 도움을 받아보시기 바랍니다.

## 3. 점수 해석 원칙

1. `정상`, `비정상`, `상위 몇 %`, `평균 이상`이라는 표현을 사용하지 않는다.
2. 규준이 없으므로 절대적인 성격 판정을 하지 않는다.
3. 7개 영역을 동일한 0~100 환산점수로 표시하되 이용자 본인의 프로필 안에서 상대적으로 비교한다.
4. 점수 차이가 작으면 순위를 강조하지 않는다.
5. 경계점수 하나로 결과 문장이 급격히 달라지지 않도록 완충구간을 둔다.
6. 높은 점수와 낮은 점수 모두 강점과 부담 가능성을 함께 설명한다.
7. 단일 영역보다 영역 간 관계를 중심으로 해석하되, 한 사람을 고정 유형으로 명명하지 않는다.

## 4. 결과 생성 구조

결과는 완성문 하나를 선택하지 않고 검토된 모듈을 규칙에 따라 조립한다.

1. 결과 이용 안내
2. 전체 프로필 요약
3. 두드러진 기질 경향
4. 두드러진 성격 경향
5. 영역 간 균형과 차이
6. 핵심 2영역 조합 최대 2개
7. 주요 3영역 조합 최대 1개
8. 생활·관계·학습·업무 맥락 해석
9. 자기점검 질문
10. 결과 한계와 전문기관 안내

같은 뜻의 모듈은 중복 출력하지 않으며, 조합 해석이 기본 해석과 충돌하면 더 구체적인 조합 해석을 우선한다.

## 5. 결과 참고도

결과 참고도는 응답자의 진실성이나 성실성을 판단하지 않는다. 이번 응답에서 영역 간 차이를 얼마나 세밀하게 읽을 수 있는지를 안내하는 보조 정보로만 사용한다.

참고도를 낮추는 조건 예시:

- 모든 문항에 같은 응답을 선택
- 대부분의 문항에서 한 응답값만 반복
- 서로 반대되는 의미의 문항에 동일한 방향으로 응답
- 완료 시간이 내부 최소 기준보다 현저히 짧음
- 7개 영역 점수 차이가 거의 없음

표시 예시:

> 여러 문항에서 비슷한 응답이 반복되어 영역 간 차이가 작게 나타났습니다. 이번 결과는 뚜렷한 유형 판정보다 전반적인 경향을 살펴보는 자료로 활용해 주세요.

## 6. 문항 개발 기준

- 영역별 예비문항을 충분히 작성한 뒤 편집 검토를 거쳐 공개 문항을 선정한다.
- 한 문항에는 하나의 행동·감정·판단만 묻는다.
- 정답처럼 보이는 사회적 바람직성 문항을 줄인다.
- 이중부정과 복잡한 역문항은 사용하지 않는다.
- 생활, 관계, 결정, 학습·업무 등 맥락을 분산한다.
- 정식 TCI 문항을 복제하거나 가깝게 바꾸어 사용하지 않는다.
- 성인용으로 먼저 개발하고 청소년용은 별도 문항과 별도 해석으로 개발한다.

## 7. 내부 QA 기준

공개 전 다음 항목을 자동 또는 수동으로 확인한다.

- 문항 ID 중복 없음
- 영역별 문항 수와 정·역방향 균형
- 허용 응답값 외 입력 차단
- 역채점 계산 검증
- 영역별 최소·최대 점수 검증
- 모든 문항 1점, 3점, 5점 응답 결과 확인
- 주요 극단 프로필과 균형 프로필 결과 확인
- 해석 모듈 중복·모순·과잉 출력 검사
- 결과 보고서 최대 길이 제한
- 전문기관 링크와 외부 링크 확인
- 모바일, 키보드, 화면낭독기 기본 접근성 확인
- 새로고침, 뒤로가기, 중도 이탈, 미응답 처리 확인
- 응답과 결과가 외부 서비스로 전송되지 않는지 확인

## 8. 개인정보 처리

- 이름, 이메일, 전화번호, 계정 정보를 받지 않는다.
- 응답과 결과는 브라우저 안에서 계산한다.
- 응답값과 점수를 운영자 서버나 외부 분석 서비스로 전송하지 않는다.
- 문항 진행을 유지할 필요가 있으면 `sessionStorage`만 사용하고 종료 시 삭제되도록 한다.
- 외부 분석도구, 광고 스크립트, 외부 AI API는 초기 버전에 연결하지 않는다.
- 호스팅 사업자가 보안과 서비스 제공을 위해 일반적인 접속 기록을 처리할 수 있다는 점은 개인정보 안내에 구분하여 표시한다.

## 9. 전문기관 안내

결과 하단은 두 갈래로 구분한다.

### 정식 검사와 전문 해석이 필요한 경우

- 검사 실시 자격을 갖춘 상담·심리 전문가
- 정식 검사 제공기관
- 검사 종류, 실시 자격, 비용, 결과 해석 방식을 이용 전 확인

### 정서적 어려움이나 위기 지원이 필요한 경우

- 국가정신건강정보포털
- 국립정신건강센터
- 보건복지상담센터 129
- 자살예방상담전화 109

기질·성격 점검 자체를 위기선별검사처럼 보이게 하지 않도록 전문검사 안내와 위기지원을 시각적으로 분리한다.

## 10. 버전 운영

- `0.1 beta`: 영역 정의, 기본 채점, 핵심 해석 모듈, 공개 고지
- `0.2 beta`: 조합 규칙과 결과 충돌 검사 확대
- `0.3 beta`: 문항 문장과 모바일 사용성 수정
- `0.4 beta`: 전문 내용 검토 반영
- `1.0`: 충분한 문서화와 반복 검토 후 안정판 표시

통계적 검증자료가 없는 동안에는 버전이 올라가더라도 `표준화`, `검증 완료`, `정확한 성격 판정`이라는 표현을 사용하지 않는다.
