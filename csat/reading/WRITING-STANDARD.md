- 문서 성격: 독서 페이지 제작의 단일 최상위 기준문서
- 적용 범위: `csat/reading/` 아래 모든 수능·모의평가 독서 본문과 관련 Note
- 기준 버전: 1.0
- 확정일: 2026-07-24
- 변경 원칙: 사용자의 명시적 지시가 있을 때만 이 문서를 수정한다. 작업 중 임의로 기준을 추가하거나 축소하지 않는다.

---

## 1. 작업 시작 전 의무 절차

새 글을 쓰기 전에 반드시 다음 순서를 지킨다.

1. 이 문서 `csat/reading/WRITING-STANDARD.md`를 다시 읽는다.
2. 핵심 기준 페이지를 다시 읽는다.
   - 우선 기준: `csat/reading/2022/september/04/index.html`
   - 최신 완성형 페이지가 별도로 지정되면 그 페이지도 함께 읽는다.
3. 대상 시험의 공식 문제지와 공식 정답·해설을 확인한다.
4. 독서 문항 범위와 문학 문항 범위를 구분한다.
5. 정답, 지문 주제, 문단 수, 문항 유형을 확정한 뒤에만 작성한다.
6. 선택률·정답률·배점은 공식 자료나 검증된 자료가 있을 때만 넣는다. 확인되지 않은 수치는 만들지 않는다.
7. 기존 파일이 있는지 확인하고, 수정할 때는 반드시 최신 SHA를 다시 가져온다.

이 절차를 거치지 않은 상태에서는 본문을 작성하지 않는다.

---

## 2. 글의 목표

이 페이지는 지문 내용을 짧게 요약하는 해설지가 아니다.

독자가 다음 능력을 실제로 기르도록 구성한다.

- 중심문장을 찾는 능력
- 중심문장까지 이어지는 논리 구조를 파악하는 능력
- 문장의 기능과 문단의 설명 방식을 구분하는 능력
- 전체 글에서 각 문단의 역할을 파악하는 능력
- 앞 문단과 다음 문단의 유기적 연결을 해석하는 능력
- 어려운 문장을 쉬운 말로 다시 이해하는 능력
- 선택지가 지문을 어떻게 바꾸어 틀리게 만드는지 판단하는 능력

모든 글은 처음 작성할 때부터 완성형이어야 한다. 나중에 보충한다는 전제로 빈 영역, 임시 링크, 짧은 요약형 문단을 남기지 않는다.

## 3. 페이지 상단 필수 구성

페이지 상단에는 다음 요소를 둔다.

1. 경로 표시
   - 수능 국어 → 독서 → 학년도·시험·문항
2. 정확한 H1
   - 지문의 핵심 주제를 쉬운 말로 표현한다.
3. 짧은 안내문
   - 문제를 먼저 풀고 문단의 논리를 직접 확인하는 학습 페이지임을 밝힌다.
4. 모든 해설 펼치기·접기 버튼
5. 어려운 말 말풍선 안내
6. 문제지 선풀이 안내 박스
7. 문제지에서 먼저 표시할 항목 체크리스트

정답은 채점 전에는 노출하지 않는다.

## 4. 문단별 필수 해설 구조

모든 문단은 아래 순서를 빠짐없이 따른다.

### 4-1. 문단을 읽기 전 질문

단순히 “중심문장을 찾으세요”라고 쓰지 않는다.

독자가 무엇을 기준으로 읽어야 하는지 구체적인 질문을 제시한다.

예:

- 이 문단은 문제를 제기하는가, 해결 원리를 제시하는가?
- 앞 문장의 사례가 뒤의 일반 원리로 어떻게 이어지는가?
- 비교되는 두 대상의 기준은 무엇인가?

### 4-2. 중심문장 제시

- 문단의 중심문장을 분명히 제시한다.
- 바로 아래에 그 문장을 중심문장으로 선정한 이유를 설명한다.
- 첫 문장이라는 이유만으로 중심문장으로 처리하지 않는다.
- 문단 전체 내용을 포괄하고 뒤 문장들을 설명할 수 있는지를 근거로 판단한다.

### 4-3. 중심문장까지 이어지는 논리 구조

문단의 문장들이 중심문장에 도달하는 과정을 흐름도로 제시한다.

가능한 기능 표지:

- 화제 제시
- 문제 제기
- 개념 정의
- 기준 제시
- 원인
- 과정
- 결과
- 사례
- 비교·대조
- 반론
- 제한
- 결론

단순 문장 요약의 나열이 아니라, 왜 다음 문장이 필요한지를 보여 준다.

### 4-4. 주요 문장별 기능 해설

원문 순서를 유지하면서 주요 문장을 해설한다.

각 문장마다 다음 중 무엇을 하는지 밝힌다.

- 앞 문장의 내용을 구체화함
- 개념을 정의함
- 원인을 설명함
- 예시를 제시함
- 예외나 한계를 제시함
- 결론을 도출함
- 다음 문단의 질문을 준비함

원문을 과도하게 길게 인용하지 않고, 필요한 핵심 표현만 제시한 뒤 해설한다.

### 4-5. 문단의 설명 방식

해당 문단이 어떤 방식으로 전개되는지 명시한다.

예:

- 정의
- 비교·대조
- 원인과 결과
- 문제와 해결
- 원리와 적용
- 주장과 근거
- 일반 원칙과 사례
- 통념과 반박
- 과정 설명

### 4-6. 전체 글에서 문단의 역할

문단을 고립시켜 설명하지 않는다.

각 문단이 전체 글에서 맡는 역할을 밝힌다.

예:

- 논의 대상 도입
- 기본 전제 설정
- 핵심 개념 정의
- 기존 견해의 문제 제기
- 해결 원리 제시
- 사례 적용
- 한계 또는 예외 제시
- 결론 정리

### 4-7. 앞뒤 문단의 유기적 연결

각 문단 끝에 다음 세 항목을 둔다.

1. 앞 문단에서 이어지는 점
2. 이 문단이 새로 하는 일
3. 다음 문단으로 넘어가는 이유

첫 문단은 앞 문단 연결을 생략할 수 있고, 마지막 문단은 다음 문단 연결 대신 전체 결론과의 관계를 설명한다.

### 4-8. 쉬운 해석과 이해 예시

해당 문단 전체를 고등학생이 이해할 수 있는 쉬운 말로 다시 풀어 쓴다.

필요할 때 다음을 사용한다.

- 생활 장면
- 간단한 숫자 예시
- 친숙한 비교
- 잘못 이해하기 쉬운 반례
- 개념 차이를 보여 주는 짧은 상황

지문에 없는 예시는 반드시 “이해를 위한 예”라고 구분한다.

### 4-9. 문제 해결 팁

해당 문단이 실제 문항에서 어떻게 변형되는지 설명한다.

공통 문구를 반복하지 않고 지문별 함정을 구체적으로 제시한다.

예:

- 주체를 바꾼 선택지
- 원인과 결과를 뒤집은 선택지
- 조건과 결론을 바꾼 선택지
- 일부 범위를 전체로 확대한 선택지
- 시간 순서를 바꾼 선택지
- 비슷한 개념을 서로 바꾼 선택지
- 보기의 수치를 적용할 때 먼저 계산해야 할 항목

## 5. 어려운 말 말풍선 기준

말풍선은 필수 기능이다. 문단 해설과 문제 해설에서 어려운 표현을 즉시 확인할 수 있어야 한다.

### 5-1. 적용 대상

- 철학·법률·경제·과학의 전문 개념
- 뜻을 추측하기 어려운 한자어
- 일상어와 지문 속 의미가 다른 단어
- 문장 이해에 결정적인 추상어
- 학생이 범위나 주체를 혼동하기 쉬운 표현

### 5-2. 설명 원칙

- 고등학생이 바로 이해할 수 있는 한 문장으로 쓴다.
- 사전식 어려운 정의를 다시 옮기지 않는다.
- 말풍선은 단어의 즉각적인 뜻만 설명한다.
- 문단 전체 의미는 별도의 쉬운 해석에서 설명한다.

### 5-3. 작동 방식

- PC: 마우스 올리기와 키보드 초점으로 열린다.
- 모바일: 단어를 누르면 열린다.
- 다른 단어를 누르면 기존 말풍선은 닫힌다.
- 화면의 다른 곳을 누르면 모두 닫힌다.
- 단어에는 점선 밑줄을 사용하되 본문 흐름을 해치지 않는다.

기준 구현:

```html
<button class="word-tip" type="button" data-tip="쉬운 뜻풀이">어려운 말</button>
```

## 6. 글 전체 해설 필수 구성

문단별 해설이 끝난 뒤 다음 내용을 둔다.

### 6-1. 전체 글의 핵심 질문

글 전체가 답하려는 질문을 한 문장으로 제시한다.

### 6-2. 전체 논리 지도

도입부터 결론까지의 관계를 화살표 흐름도로 정리한다.

### 6-3. 문단 관계표

각 문단의 중심 내용, 설명 방식, 전체 글에서의 역할을 표로 비교한다.

### 6-4. 핵심 개념 비교

서로 혼동하기 쉬운 개념이 있을 때만 표나 카드로 구분한다.

### 6-5. 문항 연결 지도

각 문항이 요구하는 독해 능력을 표시한다.

예:

- 내용 일치
- 전개 방식
- 보기 적용
- 인과 추론
- 개념 비교
- 어휘의 문맥적 의미

## 7. 문항별 채점과 해설 기준

### 7-1. 채점 전

- 정답을 화면에 미리 표시하지 않는다.
- 사용자가 선택한 뒤 채점 버튼을 눌러야 정답과 해설이 열린다.

### 7-2. 채점 후

각 문항은 다음을 포함한다.

1. 정답 여부
2. 확정 정답
3. 문항이 확인하는 독해 능력
4. 정답의 근거가 있는 문단과 문장
5. ①～⑤ 모든 선택지의 판단
6. 오답이 지문을 어떻게 바꾸었는지 설명
7. 필요한 경우 보기 적용 순서나 계산 과정

“주체·범위·원인을 확인하세요”처럼 모든 글에 공통으로 쓸 수 있는 짧은 문구만으로 끝내지 않는다.

### 7-3. 통계

- 공식 또는 검증된 선택률·정답률만 사용한다.
- 출처를 확인할 수 없는 수치는 넣지 않는다.
- 수치가 없으면 “확인되지 않은 선택률은 표시하지 않았다”고 명시할 수 있다.

## 8. Note 운영 기준

문단 해설을 다시 요약한 짧은 글은 별도 Note로 만들지 않는다.

### 8-1. 본문에 이어 붙이는 내용

- 해당 문단을 이해하기 위한 보충 설명
- 한 문항에만 필요한 적용 설명
- 본문에서 이미 충분히 설명할 수 있는 개념
- 쉬운 예시와 오해하기 쉬운 지점

### 8-2. 독립 Note 생성 조건

다음 중 하나 이상을 만족할 때만 생성한다.

- 여러 연도·여러 지문에서 반복되는 개념
- 독립 검색어로 가치가 있는 개념
- 본문에 넣으면 흐름을 크게 방해할 정도로 긴 내용
- 다른 글에서도 공통으로 연결할 수 있는 독해 도구

### 8-3. 금지

- 지문마다 정해진 수의 Note를 자동 생성하지 않는다.
- 존재하지 않는 링크를 넣지 않는다.
- `href="#"` 또는 “발행 후 교체” 같은 임시 링크를 넣지 않는다.
- 본문과 Note에 같은 설명을 반복하지 않는다.

## 9. 디자인·상호작용 기준

- 기존 독서 페이지의 색상과 카드 디자인을 유지한다.
- 모바일에서 표가 잘리지 않도록 가로 스크롤을 제공한다.
- 펼치기·접기 버튼이 실제로 작동해야 한다.
- 각 문항은 독립적으로 선택하고 채점할 수 있어야 한다.
- 채점 전 해설 영역은 숨긴다.
- 말풍선은 키보드와 모바일에서도 작동해야 한다.
- 작은 내부 박스를 과도하게 늘리지 않는다.
- 흐름도, 비교표, 역할표는 실제 논리 관계를 보여 줄 때만 사용한다.

## 10. 발행 전 최종 검수표

다음 항목을 모두 통과해야 완성으로 판정한다.

### 문단 해설

- [ ] 모든 문단에 읽기 전 질문이 있다.
- [ ] 모든 문단에 중심문장과 선정 이유가 있다.
- [ ] 중심문장까지 이어지는 논리 구조가 있다.
- [ ] 주요 문장의 기능을 원문 순서대로 설명했다.
- [ ] 문단의 설명 방식을 밝혔다.
- [ ] 전체 글에서 문단의 역할을 밝혔다.
- [ ] 앞뒤 문단 연결을 해석했다.
- [ ] 쉬운 해석이나 이해 예시가 있다.
- [ ] 문단별 문제 해결 팁이 있다.
- [ ] 어려운 말 말풍선이 필요한 곳에 적용됐다.

### 글 전체

- [ ] 전체 글의 핵심 질문이 있다.
- [ ] 전체 논리 지도가 있다.
- [ ] 문단 관계표가 있다.
- [ ] 문항 연결 지도가 있다.

### 문항 해설

- [ ] 채점 전 정답이 보이지 않는다.
- [ ] 모든 문항을 독립 채점할 수 있다.
- [ ] ①～⑤ 모든 선택지를 분석했다.

### 기술 검수

- [ ] canonical 주소가 정확하다.
- [ ] HTML 태그와 스크립트가 정상 종료된다.
- [ ] 펼치기·접기 기능이 작동한다.
- [ ] 말풍선이 PC·모바일·키보드에서 작동한다.
- [ ] 모바일에서 표와 버튼이 정상 표시된다.
- [ ] 존재하지 않는 Note 링크가 없다.
- [ ] 진행표는 실제 완성 후에만 완료로 바뀐다.

## 11. 완성 판정

다음 조건을 모두 만족해야 “완료”라고 보고한다.

1. 본문이 이 문서의 모든 필수 항목을 갖춘다.
2. 정답과 문항 범위가 공식 자료와 일치한다.
3. 채점·펼치기·말풍선 기능이 정상 작동한다.
4. 실제로 생성된 Note만 연결한다.
5. 저장된 파일을 다시 읽어 canonical, 정답 배열, 스크립트 종료, 링크를 확인한다.
6. 진행표에 실제 주소를 연결한다.

위 조건 중 하나라도 빠지면 초안 또는 미완성으로 보고하며, 완료라고 표현하지 않는다.

## 12. 매 작업 시 고정 실행 문장

새 독서 글을 시작할 때 내부 작업 순서는 다음과 같이 고정한다.

> 작성기준 문서 확인 → 기준 페이지 확인 → 공식 문제·해설 확인 → 문항 범위·정답 확정 → 완성형 본문 작성 → 기능 검수 → 저장 파일 재확인 → 진행표 반영

이 문서에 이미 정해진 항목을 작업 도중 다시 협의하거나 추가 기준처럼 제안하지 않는다. 처음부터 이 기준을 모두 적용한 완성글을 작성한다.
