# 수능 국어 독서 페이지 최종 작성기준

- 문서 성격: `csat/reading/` 제작의 단일 최상위 기준문서
- 기준 버전: 1.1
- 확정일: 2026-07-24
- 변경 원칙: 사용자의 명시적 지시가 있을 때만 수정한다. 작업 중 기준을 임의로 추가·축소하지 않는다.

---

## 1. 작업 시작 전 의무 절차

새 글을 쓰기 전에 반드시 다음 순서를 지킨다.

1. `csat/reading/WRITING-STANDARD.md`를 다시 읽는다.
2. 핵심 기준 페이지 `csat/reading/2022/september/04/index.html`을 다시 읽는다.
3. 대상 시험의 공식 문제지와 공식 정답·해설을 확인한다.
4. 독서 문항 범위, 지문 주제, 문단 수, 정답, 문항 유형을 확정한다.
5. 선택률·정답률·배점은 공식 또는 검증된 자료가 있을 때만 사용한다.
6. 기존 파일 유무를 확인하고, 수정 시 최신 SHA를 다시 가져온다.
7. 관련 개념을 연결하기 전에 저장소와 블로그 내부에 이미 발행된 관련 글이 있는지 검색한다.

이 절차를 거치기 전에는 본문을 작성하지 않는다.

---

## 2. 글의 목표

이 페이지는 지문 내용을 짧게 요약하는 해설지가 아니다. 독자가 다음 능력을 실제로 기르도록 구성한다.

- 중심문장을 찾는 능력
- 중심문장까지 이어지는 논리·설명 구조를 파악하는 능력
- 문장 기능과 문단 전개 방식을 구분하는 능력
- 전체 글에서 문단의 역할을 파악하는 능력
- 문단과 문단의 유기적 연결을 해석하는 능력
- 어려운 문장을 쉬운 말로 다시 이해하는 능력
- 선택지가 지문을 어떻게 바꾸어 틀리게 만드는지 판단하는 능력

모든 글은 처음부터 완성형으로 작성한다. 빈 영역, 임시 링크, 나중에 보충할 짧은 초안을 남기지 않는다.

## 3. 페이지 상단 필수 구성

1. 경로 표시: 수능 국어 → 독서 → 학년도·시험·문항
2. 정확한 H1과 canonical
3. 문제를 먼저 풀고 논리를 직접 확인하는 페이지라는 안내
4. 모든 해설 펼치기·접기 버튼
5. 어려운 말 말풍선 안내
6. 문제지 선풀이 안내 박스
7. 문제지에서 먼저 표시할 항목 체크리스트

정답은 채점 전 노출하지 않는다.

---

## 4. 문단별 필수 해설 구조

모든 문단은 아래 항목을 빠짐없이 포함한다.

### 4-1. 문단을 읽기 전 질문

단순히 중심문장을 찾으라고 하지 않는다. 비교 기준, 원인과 결과, 문제와 해결, 원리와 적용 등 무엇을 기준으로 읽어야 하는지 구체적으로 질문한다.

### 4-2. 중심문장과 선정 이유

- 중심문장을 분명히 제시한다.
- 왜 그 문장이 문단 전체를 포괄하는지 설명한다.
- 첫 문장이라는 이유만으로 중심문장으로 처리하지 않는다.

### 4-3. 중심문장까지 이어지는 논리 구조

원문 순서를 유지하며 다음과 같은 기능을 표시한다.

- 화제 제시
- 문제 제기
- 개념 정의
- 기준·원인·과정·결과
- 사례
- 비교·대조
- 반론·제한
- 결론

문장을 요약해 나열하지 않고, 왜 다음 문장이 필요한지 보여 준다.

### 4-4. 주요 문장별 기능 해설

각 주요 문장이 앞 문장을 구체화하는지, 개념을 정의하는지, 근거·사례·예외·결론을 제시하는지 밝힌다. 원문 전체를 복제하지 않고 필요한 핵심 표현만 인용한다.

### 4-5. 문단의 설명 방식

정의, 비교·대조, 원인과 결과, 문제와 해결, 원리와 적용, 주장과 근거, 일반 원칙과 사례, 통념과 반박, 과정 설명 중 실제 전개 방식을 명시한다.

### 4-6. 전체 글에서 문단의 역할

논의 대상 도입, 기본 전제, 핵심 개념 정의, 문제 제기, 해결 원리, 사례 적용, 경계·예외, 정당화, 결론 중 해당 역할을 밝힌다.

### 4-7. 앞뒤 문단의 유기적 연결

각 문단 끝에 다음을 해석한다.

1. 앞 문단에서 이어지는 점
2. 이 문단이 새로 하는 일
3. 다음 문단으로 넘어가는 이유

첫 문단은 앞 문단 연결을 생략할 수 있고, 마지막 문단은 전체 결론과의 관계를 설명한다.

### 4-8. 쉬운 해석과 이해 예시

문단 전체를 고등학생이 이해할 수 있는 쉬운 말로 다시 풀어 쓴다. 생활 장면, 간단한 숫자, 친숙한 비교, 반례를 사용할 수 있으며 지문에 없는 예시는 반드시 `이해를 위한 예`라고 표시한다.

### 4-9. 문제 해결 팁

해당 문단이 문항에서 어떻게 변형되는지 구체적으로 설명한다.

- 주체 변경
- 원인·결과 역전
- 조건·결론 변경
- 일부를 전체로 확대
- 시간 순서 변경
- 유사 개념 바꿔치기
- 보기 적용 순서와 계산 기준

공통 문구만 반복하지 않는다.

## 5. 어려운 말 말풍선 기준

말풍선은 필수 기능이다.

### 적용 대상

- 철학·법률·경제·과학 전문 개념
- 뜻을 추측하기 어려운 한자어
- 일상어와 지문 속 의미가 다른 단어
- 문장 이해에 결정적인 추상어
- 범위와 주체를 혼동하기 쉬운 표현

### 설명 원칙

- 고등학생이 바로 이해할 수 있는 한 문장으로 쓴다.
- 사전식 어려운 정의를 반복하지 않는다.
- 말풍선은 단어의 즉각적인 뜻만 설명한다.
- 문단 전체 의미는 쉬운 해석에서 별도로 설명한다.

### 작동 방식

- PC: 마우스 올리기와 키보드 초점
- 모바일: 단어 누르기
- 다른 단어를 누르면 기존 말풍선 닫기
- 화면 다른 곳을 누르면 모두 닫기
- 점선 밑줄을 사용하되 본문 흐름을 해치지 않기

```html
<button class="word-tip" type="button" data-tip="쉬운 뜻풀이">어려운 말</button>
```

5. 각 문항이 요구하는 독해 능력을 보여 주는 문항 연결 지도

## 7. 문항별 채점과 해설 기준

### 채점 전

- 정답, 통계, 선택지 해설을 숨긴다.
- 사용자가 선택하고 채점 버튼을 눌러야 열린다.

### 채점 후

각 문항은 다음을 포함한다.

1. 정답 여부와 확정 정답
2. 문항이 확인하는 독해 능력
3. 근거 문단과 핵심 판단
4. ①～⑤ 모든 선택지 분석
5. 오답이 지문의 무엇을 바꾸었는지 설명
6. 보기 문항은 적용 순서와 판단 과정을 구체적으로 제시

### 통계

공식 또는 검증된 수치만 사용한다. 확인되지 않은 수치는 넣지 않는다.

## 8. 블로그 내부 연결과 Note 운영 기준

### 8-1. 우선순위

관련 개념을 보충할 때 다음 순서를 지킨다.

1. 저장소와 운영 블로그 안에 이미 발행된 관련 개념 글이 있는지 검색한다.
2. 실제 주소와 내용이 확인된 기존 글이 있으면 우선 연결한다.
3. 짧은 뜻만 필요하면 본문의 말풍선으로 설명한다.
4. 말풍선 한 문장으로 부족하고 본문에 모두 넣으면 흐름을 크게 방해하는 개념만 상세 Note로 작성한다.
5. 본문 설명만으로 충분하면 별도 Note를 만들지 않는다.

### 8-2. 기존 블로그 글 연결 조건

- 현재 지문 개념과 직접 관련되어야 한다.
- 실제 주소와 제목을 확인해야 한다.
- 단순히 같은 단어가 포함됐다는 이유로 연결하지 않는다.
- 링크를 열었을 때 독자의 이해가 실제로 확장되어야 한다.

### 8-3. 상세 Note 생성 조건

다음 중 하나 이상을 만족할 때만 생성한다.

- 말풍선 한 문장으로 개념의 구조를 설명하기 어렵다.
- 여러 연도·여러 지문에서 반복되는 개념이다.
- 독립 검색어로 가치가 있다.
- 본문에 넣으면 문단의 독해 흐름을 방해할 정도로 길다.
- 다른 지문에서도 재사용할 수 있는 개념 또는 독해 도구이다.

상세 Note는 개념 정의만 반복하지 않고 원리, 관계, 쉬운 예, 오해 지점, 기출 적용법을 포함한다.

### 8-4. 본문에 이어 붙이는 내용

- 해당 문단만 이해하면 되는 보충 설명
- 한 문항에만 필요한 적용 설명
- 쉬운 예와 오해하기 쉬운 지점
- 본문 안에서 충분히 설명할 수 있는 개념

### 8-5. 금지

- 지문마다 정해진 수의 Note를 자동 생성하지 않는다.
- 문단 해설을 다시 요약한 짧은 Note를 만들지 않는다.
- 존재하지 않는 링크, `href="#"`, 발행 후 교체 링크를 넣지 않는다.
- 본문과 Note에 같은 설명을 반복하지 않는다.
- 검색 결과가 없는데 관련 글이 있다고 가정하지 않는다.

## 11. 완성 판정

다음 조건을 모두 만족해야 완료라고 보고한다.

1. 모든 필수 문해력 항목을 갖춘다.
2. 문항 범위와 정답이 공식 자료와 일치한다.
3. 채점·펼치기·말풍선 기능이 정상 작동한다.
4. 내부 링크와 Note는 실제 존재하는 파일·주소만 사용한다.
5. 저장 파일을 다시 읽어 canonical, 정답 배열, 링크, 스크립트 종료를 확인한다.
6. 실제 주소를 진행표에 연결한다.

하나라도 빠지면 초안 또는 미완성으로 보고한다.

## 12. 매 작업 시 고정 실행 문장

> 작성기준 확인 → 기준 페이지 확인 → 공식 문제·해설 확인 → 문항 범위·정답 확정 → 내부 관련 글 검색 → 완성형 본문 작성 → 필요한 상세 Note만 작성 → 기능 검수 → 저장 파일 재확인 → 진행표 반영
