CPU와 GPU 병렬 연산

3D 영상은 정점과 화소의 수가 많아질수록 연산량이 크게 증가합니다. 이때 CPU와 GPU의 처리 방식 차이를 구분해야 합니다.

핵심 비교

구분CPUGPU
코어상대적으로 적고 개별 코어가 빠름수백～수천 개, 개별 코어는 상대적으로 느림
처리데이터 연산을 하나씩 순서대로 수행같은 연산을 많은 데이터에 동시에 수행하는 데 유리
강점복잡하고 다양한 연산반복되는 그래픽 연산

GPU 처리 순서

한 번의 연산에 쓰일 데이터 준비 → 데이터를 각 코어에 순차 전송 → 전체 코어에 하나의 연산 명령 전달 → 각 코어가 데이터를 동시에 연산 → 총 연산 시간 단축

‘순차 전송’과 ‘동시 연산’을 섞어 읽지 않는 것이 핵심입니다. 데이터는 각 코어로 차례로 보내지만, 연산 자체는 여러 코어에서 동시에 이루어집니다.

병목 현상

CPU의 처리 능력보다 많은 데이터가 몰리면 아직 연산되지 못한 데이터가 차례를 기다립니다. 이 대기 때문에 프레임 완성이 늦어지는 현상이 병목입니다.

수능 추론 기준

연산을 반드시 하나씩 순서대로 해야 하는 조건에서는 코어가 많아도 병렬 처리의 장점을 사용할 수 없습니다. 따라서 여러 코어와 한 코어의 총 연산 시간이 같아질 수 있습니다.

확인 문제

GPU는 코어마다 서로 다른 연산 명령을 보내는 것이 아니라 전체 코어에 하나의 연산 명령을 전달합니다.
