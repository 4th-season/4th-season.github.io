3D 합성 영상의 생성과 출력

문제를 먼저 풀고 네 문단의 중심 문장을 직접 고른 뒤, 모델링·렌더링·프레임 생성·연산 처리의 관계를 확인합니다.

해설보다 먼저 지문에 표시하세요.

‘고유한 값 설정’과 ‘관찰 시점에 따른 화솟값 결정’, ‘데이터 전송’과 ‘동시 연산’을 서로 다른 표시로 구분합니다.

문제지에서 먼저 해 볼 일




1문단

왜 3D 합성 영상에는 모델링과 렌더링이 필요한가

중심 문장 찾기

자연 영상과 3D 합성 영상의 생성 방식을 대비한 문장을 고릅니다.

1문단 원문 중심 해설

중심: 자연 영상과 달리 3D 합성 영상은 가상 물체를 만들고 화면에 표현하기 위해 모델링과 렌더링을 거친다.

① 화제 제시

3D 애니메이션은 실물 촬영처럼 보이지만 실제 촬영 영상을 그대로 표시한 것은 아닙니다.

② 두 방식의 대비

자연 영상은 촬영 결과를 표시하지만 3D 합성 영상은 모델링으로 가상 물체를 만들고 렌더링으로 화면을 생성합니다.

자연 영상: 실물 촬영 → 화면 표시 3D 합성 영상: 모델링 → 렌더링 → 화면 출력
2문단

모델링은 물체의 모양과 표면을 어떻게 정하는가

중심 문장 찾기

정점의 개수, 간격, 상대적 위치를 물체 변화와 연결합니다.

2문단 원문 중심 해설

중심: 모델링은 3차원 공간에서 물체 고유의 값을 설정하고 삼각형과 정점으로 표면을 구성하는 단계이다.

① 고유한 값 설정

모양·크기·공간적 위치·표면 특성을 설정하거나 수정합니다.

② 삼각형 그물 구조

작은 삼각형을 조합하면 복잡한 굴곡을 정밀하게 표현할 수 있습니다.

③ 정점 변화 규칙

물체가 변형되어도 정점 개수는 유지됩니다. 크기가 변하면 간격이 변하고, 이동·회전하면 간격을 유지한 채 함께 움직입니다.

④ 표면 특성 지정

각 삼각형 면에 색과 질감 같은 표면 특성을 하나씩 지정합니다.

고유한 값 설정 → 삼각형·정점 구성 → 변형에 따른 정점 조정 → 면별 표면 특성 지정
3문단

렌더링은 3차원 정보를 어떻게 화면으로 바꾸는가

중심 문장 찾기

관찰 시점, 원근감, 입체감, 화솟값의 연결을 찾습니다.

3문단 원문 중심 해설

중심: 렌더링은 모델링 데이터를 관찰 시점에 맞는 2차원 화면으로 바꾸고 화솟값을 정해 원근감과 입체감을 구현한다.

① 2차원 화면 생성

물체를 바라보는 관찰 시점을 기준으로 화면을 만듭니다.

② 화소와 화솟값

정해진 수의 화소마다 밝기와 색상 값을 부여합니다.

③ 원근감과 입체감

거리, 표면 특성, 가려짐, 조명, 명암과 그림자를 고려해 화솟값을 정합니다.

④ 프레임

모든 화소의 값이 정해지면 한 프레임이 생성되고 모니터에 표시하면 정지 영상이 됩니다.

모델링 데이터 → 관찰 시점 적용 → 화솟값 결정 → 프레임 생성
4문단

GPU는 CPU의 병목 현상을 어떻게 보완하는가

중심 문장 찾기

데이터의 순차 전송과 여러 코어의 동시 연산을 구분합니다.

4문단 원문 중심 해설

중심: CPU의 순차 연산은 많은 데이터에서 병목을 만들 수 있고, GPU는 많은 코어가 같은 연산을 동시에 수행해 시간을 줄인다.

① 연산량 증가

정점과 출력 화소가 많을수록 연산량과 시간이 증가합니다.

② CPU와 병목

처리 능력을 넘는 데이터가 순서를 기다리면 프레임 완성이 늦어집니다.

③ GPU의 코어

각 코어는 저속이지만 많은 코어가 같은 연산을 반복할 때 유리합니다.

④ 처리 순서

데이터를 각 코어에 순차 전송한 뒤 하나의 명령을 전달하면 여러 코어가 동시에 연산합니다.

데이터 증가 → CPU 대기·병목 → GPU에 데이터 순차 전송 → 하나의 명령 → 동시 연산
